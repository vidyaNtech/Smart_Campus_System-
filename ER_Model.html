2. College Management Specific Relationships

  • Management (1) → (N) Usage Report: College management reviews aggregated monthly/weekly usage reports to identify underutilized resources and track ROI.
  • Management (1) → (N) Maintenance Log: Management oversees repair costs, scheduling, and downtime for all campus resources.
  • Building (1) → (N) Resource: Resources are now grouped by Physical Buildings/Zones, allowing management to analyze efficiency on a building-by-building basis (e.g., shutting down HVAC in an empty wing).
  • Resource (1) → (N) Maintenance Log: A resource can have multiple maintenance histories impacting its availability for bookings.

3. Additional SQL Table Structure (Management Focus)


-- College Management Table
CREATE TABLE Management (
    management_id VARCHAR(50) PRIMARY KEY,
    user_id INT UNIQUE REFERENCES Users(user_id) ON DELETE CASCADE,
    oversight_level VARCHAR(50) NOT NULL -- e.g., 'Dean', 'Facilities Head', 'Provost'
);

-- Buildings Table
CREATE TABLE Buildings (
    building_id SERIAL PRIMARY KEY,
    building_name VARCHAR(100) NOT NULL,
    zone VARCHAR(50) NOT NULL
);

-- Maintenance Logs Table
CREATE TABLE Maintenance_Logs (
    log_id SERIAL PRIMARY KEY,
    resource_id INT REFERENCES Resources(resource_id) ON DELETE CASCADE,
    issue_description TEXT NOT NULL,
    cost DECIMAL(10, 2),
    start_date DATE NOT NULL,
    end_date DATE,
    status VARCHAR(20) CHECK (status IN ('Scheduled', 'In Progress', 'Completed'))
);

-- Usage Reports Table (Aggregated Analytics for Management)
CREATE TABLE Usage_Reports (
    report_id SERIAL PRIMARY KEY,
    resource_id INT REFERENCES Resources(resource_id) ON DELETE CASCADE,
    month_year VARCHAR(7) NOT NULL, -- Format: YYYY-MM
    utilization_percentage DECIMAL(5, 2) NOT NULL,
    total_conflicts_resolved INT DEFAULT 0,
    generated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
